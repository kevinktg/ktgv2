# Plan 02-02 — Blog routes: index scale, not-found, featured fallbacks

<objective>
Ensure the blog index can scale beyond the default first page of posts when needed, keep `[slug]` stable with `notFound()`, and make featured-image absence explicit (placeholder already partially via `getFeaturedImage` consumers — verify UX).
</objective>

## must_haves

- Visitor can browse `/blog` with **documented** behaviour when WordPress returns more posts than one REST page (pagination **or** increased `per_page` with comment in code).
- Unknown slug → **`notFound()`** path unchanged or improved (no 500).
- Featured media: when `_embed` missing, UI still renders (existing fallback URLs in JSON-LD — verify).
